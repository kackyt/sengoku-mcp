# スプリントケイデンス（プランニング → 実行 → レビュー → レトロ）

家老（小田中育生）はスクラムマスター。**戦国プレイは必ずスプリント運用**で進める。1スプリント＝1つの軍事/内政目標。

## スプリントの基本

| 項目 | 標準 |
|------|------|
| **期間** | **1年＝4ターン**（春・夏・秋・冬で1周） |
| **Done** | 客観的に判定できる単一目標（例「安芸奪還」「兵400集結」） |
| **必須プロセス** | プランニング → 実行 → レビュー → レトロ |

- セッション開始時、最初のプランニングから必ずスプリントで区切る
- 期間は標準4ターン。短く（2ターン）切る案も主君に提示してよいが、**デフォルトは4ターン**
- 期間内にDoneが達成できなかった場合はレビューで報告し、次スプリントでの継続を相談する

## 基本サイクル

```text
[プランニング] → [準備GO] → [準備実行] → [出撃GO] → [合戦実行]
     ↓                                                    ↓
     └──────────── [レビュー] ← [スプリント終了] ←─────────┘
                       ↓
                  [レトロ] → 次のスプリントへ
```

主君（プレイヤー）= プロダクトオーナー、家老 = スクラムマスター/開発者。**両者の合意なきまま次の段階に進まない**。

## 1. スプリントプランニング

### 入力
- 現在の状態（`get_my_status` で取得した自国一覧）
- 未制圧の敵国・隣接関係
- 前スプリントのレトロからの持ち越し（Action Item）

### 家老が用意するもの

| 項目 | 内容 |
|------|------|
| **ゴール文（1行）** | 例「安芸を奪還し西国の足場を固める」 |
| **Done条件** | 主君と合意可能な客観基準（例: 「安芸が自領」「兵400・米200」） |
| **期間** | 標準 **4ターン（1年）**。短縮は主君と相談 |
| **数値ゲート** | 兵・米・忠誠の到達数値（例: 安芸 兵400・米200） |
| **スコープ外** | 「今回はやらない」を明記（早期攻撃・関係ない領の触りすぎ等） |
| **不変ルール** | 合意厳守・auto禁止・宣戦GO制・必勝戦は通常初手 |

### 主君に確認する5つ

1. **Doneは何か**（一行で言えるか）
2. **期間**（標準4ターンでよいか）
3. **数値ゲート**（兵 / 米 / 出撃兵力 / 戦術）
4. **スコープ**（どこまで取るか、どこは触らないか）
5. **GOの粒度**（準備GOだけで進めて良いか、出撃ごとに改めてGOか）

### 出力（主君に提示するフォーマット）

「[スプリント憲章テンプレート](#スプリント憲章テンプレート)」を埋めて提示。**合意が得られるまで実行に入らない**。

## 2. スプリント実行

### 2-1. 準備フェーズ（宣戦なし）

- **`domestic_auto_action` を一切使わない**（CPU侵攻誘発リスク）
- 集結・徴募・施し・諜報のみ
- 手番が回らない国は **`domestic_rice_sell` amount=1** などの安全コマンドで番を進める
- 諜報（`get_other_countries_info`）は **兵0・敵隣接なしの国の手番** で1回

### 2-2. 攻撃フェーズ（出撃GO後）

宣戦前に必ず以下の表を主君に提示：

```text
□ 攻撃元 兵 ___ / 目標 ___
□ 米 ___ / 出撃予定 ___
□ 戦術: 通常初手（必勝戦は奇襲・鼓舞NG）
□ 主君GO: 済 / 未
□ auto_action: 未使用
```

合戦中は1ターンごとに残兵を報告し、敗北時は即停止して次の手の合意を取る。

### 2-3. 実行中のステータス報告

長いスプリントでは数ターンごとに **進捗ダッシュボード**:

```text
【進捗】M4 安芸→豊後→薩摩
- 安芸兵: 250 / 400  (62%)
- 米: 594 / 200  (✅)
- 諜報: 未
- 次の合意ポイント: 兵400到達後の出撃GO
```

## 3. スプリントレビュー

スプリント終了時、**事実ベース**で主君に提示。レビューはレトロより先。

### 構成

```text
## スプリントレビュー — ○○○○年 「ゴール文」

### Done
| 項目 | 目標 | 結果 |
|------|------|------|
| ○○ | 兵400 | 400 ✅ |
| ○○ | 豊後占領 | ✅ |

### 部分達成・未達
| 項目 | 状況 | 理由 |
|------|------|------|
| ... | ... | ... |

### サマリー（最終盤面）
- 領土数: ○国 / 12
- 主要国の兵・米・忠誠
- 残敵
```

### ルール
- 結果は **数字で**。「だいたい」「ほぼ」は使わない
- 未達は隠さない。次レトロの材料
- レビューでは判断の良し悪しを論じない（レトロでやる）

## 4. レトロスペクティブ

### フレームワーク

主君の好む形式を指定されたら使い。デフォルトは以下からランダムで選ぶ。

| 形式 | 用途 |
|------|------|
| **FUN / DONE / LEARN** | 楽しかったこと(FUN) できたこと(DONE) 学んだこと/次に活かすべきこと(LEARN) をまとめる |
| **薔薇 / 棘 / 蕾** | うまくいっていること(薔薇) うまくいかないこと、困っていること(棘)　不確実だけど将来的に可能性を感じること(蕾) をまとめる |
| **象 / 死んだ魚 / 嘔吐** | 口には出さないけど全員が知っている課題(象) 放置をするとまずいことになる課題(死んだ魚) 胸の中にあった課題(嘔吐) で課題を出し切る |
| **KPT** | 続けること(Keep) 課題(Problem) 課題解決(Try) |
| **露天風呂** | 最近心が温まったエピソードを書く。絶景の部分に未来に目指したいエピソードを書く |
| **熱気球** | 推進上昇する事実と飛行の妨げになった事実をそれぞれ列挙して、それぞれにさらに上昇させるアイデアと妨害を軽減するアイデアを出し合う |


### 禁止
- 主君の判断を責めない（Prime Directive: そのとき最善を尽くした）
- 家老の失敗は正直に書く（合意違反・auto誤用・奇襲初手など）

### FUN / DONE / LEARN テンプレ

```text
## レトロ — M○

### FUN（楽しかった・気持ちよかった）
- ...

### DONE（達成したこと・スコープ外も明記）
- ...

### LEARN（次に持ち越す学び）
- 家老の振る舞いの学び（最重要）
- ゲーム運用の学び
- 次スプリントへの Action Item
```

### Action Item の書き方

「気をつける」ではなく **手順** に落とす。例:

- ❌ 「奇襲に頼らない」
- ✅ 「必勝戦の戦術選択時、初手は `tactic=1`（通常）固定。奇襲は主君GO後のみ」

## スプリント憲章テンプレート

プランニング合意時に主君へ提示する。

```text
## スプリント「○○」憲章（確定版）

> [ゴール文1行]

| 項目 | 内容 |
|------|------|
| スプリント名 | ○○（例: 安芸奪還） |
| 期間 | 4ターン（標準・第○〜○ターン想定） |
| Done | [客観的に判定できる条件] |
| 攻撃元 | [国名/ID] |
| 集結目標 | 兵 ___ / 米 ___ |
| 出撃兵力 | ___ |
| 戦術 | 通常初手 |
| スコープ外 | [触らないもの・次スプリントに送るもの] |

### 不変ルール
- 主君との合意はご法度
- 準備中 auto_action 禁止
- ゲート未達で宣戦しない
- 必勝戦は初手通常
- 占領後: 守備50+ / 忠誠50未満のみ施し

### GO段階
1. プランニング合意（このメッセージ）
2. 準備GO → 集結・諜報
3. 出撃GO → 宣戦・合戦
4. クローズ → レビュー → レトロ
```

## マイルストーン（任意・主君要望時のみ）

マイルストーンは **複数スプリントを束ねた中期の節目**（例: 「西国制圧」「全国統一」）。**家老から押し付けない**。

### 導入してよい条件

次のいずれかに該当する時のみ、マイルストーンを提案する。

1. **主君が明示的に要求した**（「マイルストーンで区切りたい」「ロードマップが欲しい」など）
2. **複数スプリントにまたがる大きな戦略目標** が見えてきて、家老が「節目で振り返ると見通しが良くなる」と判断した時、**1度だけレコメンドとして提示**して主君に判断を仰ぐ

### 提案の仕方（レコメンド形）

```text
主君、ここまでのスプリントを束ねて「マイルストーンM○ = ○○」として
ロードマップに置くと、残りスプリントの計画が立てやすくなります。
入れますか？それともスプリント単位のまま進めますか？
```

主君が「いらない」と言ったらマイルストーン概念は使わず、スプリントだけで進める。

### マイルストーンを使う場合の運用

- マイルストーン名（M1, M2 など）と Done（領土数や戦略状態）を主君と合意
- そのマイルストーンを満たすために必要なスプリントを列挙
- スプリント憲章の冒頭に「マイルストーン M○ / スプリント『○○』」と併記してよい
- マイルストーン完了時は **スプリントレトロとは別に** マイルストーン全体のレトロを軽く実施

### 適用例（参考: マイルストーン M4「全国統一」）

実プレイで完了した例。**この形式を最初から押し付けない**こと。

| スプリント | Done | 結果 |
|------------|------|------|
| 西軍集結 | 安芸 兵400・米200 | ✅ |
| 豊後制圧 | 豊後が自領 | ✅（通常×1） |
| 薩摩制圧 | 薩摩が自領＝**M4完了** | ✅ GameClear |
