# コードレビュースキル

このスキルは `main` ブランチとの差分を解析し、`.rulesync/rules/project.md` の規約に照らした**敵対的レビュー**を実施して、指摘をMarkdownにまとめます。

## 実行フロー

### 1. 差分の取得

```bash
# 現在のブランチ名を確認
git rev-parse --abbrev-ref HEAD

# mainとの差分を取得（コンテキスト付き）
git diff main...HEAD
```

差分が大きい場合はファイル単位で分割して取得：

```bash
# 変更されたファイル一覧
git diff main...HEAD --name-only

# ファイルごとの差分
git diff main...HEAD -- <filepath>
```

### 2. 規約の確認

`.rulesync/rules/project.md` を `view_file` で読み込み、以下の規約軸を把握する。


### 3. 過去のレビュー結果の確認

`review_<ブランチ名>.md` が存在すれば `view_file` で読み込み、過去のレビュー結果を確認する。

### 4. 敵対的レビューの実施

差分をファイルごとに解析し、下記のチェック観点で**厳しく、妥協なく**指摘する。
些細な違反でも見逃さないこと。

過去のレビューを読み込んだ場合は、過去の指摘項目が修正されているかをチェックする。


| 重要度 | チェック観点 | 根拠（rules/project.md） |
|--------|-------------|--------------------------|
| 🔴 重大 | エンティティへの参照をIDではなくポインタ・参照で保持している | ID参照の原則 |
| 🔴 重大 | ドメイン層・インフラ層で `anyhow` を直接使っている（`thiserror` を使うべき） | エラーハンドリング規約 |
| 🔴 重大 | アプリ層がインフラ層の具体型に直接依存している（Trait経由でない） | DI原則 |
| 🔴 重大 | ロジック変更にテストが伴っていない | テスト戦略 |
| 🔴 重大 | **[アーキテクチャ境界違反]** ドメイン層（`engine/src/domain/`）がインフラ層（`infrastructure/`）のモジュールを `use` している | オニオンアーキテクチャ：依存は内側に向かう |
| 🔴 重大 | **[アーキテクチャ境界違反]** ドメイン層がアプリケーション層（`engine/src/application/`）のモジュールを `use` している | オニオンアーキテクチャ：ドメインはすべての層から独立 |
| 🔴 重大 | **[アーキテクチャ境界違反]** Repository の具象実装が `infrastructure/src/persistence/` 以外（例: `engine/src/domain/`）に配置されている | ヘキサゴナルアーキテクチャ：ポート（Trait）はドメイン、アダプタ（実装）はインフラ |
| 🔴 重大 | **[アーキテクチャ境界違反]** Usecase/Application Service が `engine/src/application/usecase/` 以外に実装されている | オニオンアーキテクチャ：Usecaseの配置規約 |
| 🔴 重大 | **[アーキテクチャ境界違反]** ドメインロジック(収入、イベント、成否判定など)がcliなどアプリケーションで実装されている | オニオンアーキテクチャ：依存は内側に向かう |
| 🔴 重大 | **[DDDドメインモデル違反]** ビジネスロジック（不変条件の保護、状態遷移）がドメインモデル外（Usecase や infrastructure）に漏れている | DDD：ドメインモデルが不変条件を内包すること |
| 🔴 重大 | **[DDDドメインモデル違反]** 複数のエンティティ/集約にまたがるロジックが Usecase に直接記述されており、Domain Service として分離されていない | DDD：Domain Service（`engine/src/domain/service/`）を使うこと |
| 🔴 重大 | **[DDDドメインモデル違反]** Repository の Interface（Trait） が `engine/src/domain/repository/` 以外に定義されている | DDD：RepositoryインターフェースはDomain層が所有する |
| 🔴 重大 | **[ヘキサゴナルアーキテクチャ違反]** プレゼンテーション層（`mcp-server/src/presentation/`）が Usecase を経由せずドメインモデルや Repository を直接呼び出している | ヘキサゴナル：外部からの入力はUsecase（ポート経由）でのみ受け付ける |
| 🔴 重大 | **[Composition Root違反]** DI（依存の組み立て）が `mcp-server/src/main.rs` 以外で行われている | Composition Root原則 |
| 🟡 警告 | 日本語コメントがない、または意図がわからないコメントのみ | コードスタイル規約 |
| 🟡 警告 | 不必要な `.clone()` がある | メモリライフサイクル規約 |
| 🟡 警告 | `Box<dyn Trait>` のライフサイクルが不明瞭 | メモリライフサイクル規約 |
| 🟡 警告 | SOLID原則違反（単一責任・開放閉鎖など） | SOLID/DRY原則 |
| 🟡 警告 | DRY違反（重複ロジック） | SOLID/DRY原則 |
| 🟡 警告 | プリミティブ型（`i32`, `String` 等）を直接フィールドに使用している | Newtypeパターンの徹底 |
| 🔴 重大 | **[Amount型規約違反]** `DisplayAmount` をドメインロジックや計算式の中で直接使用している | Amount型規約：DisplayAmountは表示専用 |
| 🔴 重大 | **[Amount型規約違反]** `Amount` / `DisplayAmount` を介さず `.value()` で生値を直接計算・比較している | Amount型規約：計算・比較はAmountで完結させる |
| 🟡 警告 | `Amount::new(0)` を使用している（`Amount::zero()` を使うべき） | Amount型規約 |
| 🟡 警告 | `.value() == 0` で判定している（`.is_zero()` を使うべき） | Amount型規約 |
| 🟡 警告 | `Amount` メソッド（`add`, `sub`, `mul_percent`等）で代替可能な計算を `.value()` 経由で行っている | Amount型規約 |
| 🟡 警告 | **[DDDドメインモデル]** DTO（`engine/src/application/dto/`）とドメインモデルの境界が曖昧（Entityが直接プレゼンテーション層に返される等） | DDD：DTO変換はApplication層で行う |
| 🟡 警告 | **[DDDドメインモデル]** 集約ルート（Aggregate Root）を経由せずに集約内部のエンティティを外部から直接変更している | DDD：集約の整合性境界の維持 |
| 🔵 提案 | `cargo clippy` / `cargo fmt` で検出される軽微な問題 | コードスタイル規約 |

### 5. レビュー結果のMarkdown出力

以下のフォーマットで `review_<ブランチ名>.md` をリポジトリルートに生成する。
過去のレビュー結果を読み込んだ場合は、修正されているかどうかのチェックを出力。

```markdown
# コードレビュー: <ブランチ名>
> レビュー日時: <日時>
> 対象: `git diff main...<ブランチ名>`

## サマリー

| 重要度 | 件数 |
|--------|------|
| 🔴 重大 | N件 |
| 🟡 警告 | N件 |
| 🔵 提案 | N件 |

**総評**: （全体的な所感を2〜3文で厳しく記述）

---

## 指摘事項

### 🔴 [重大] `path/to/file.rs` - 指摘タイトル

**問題**:
（何が問題なのかを具体的に説明）

**該当コード**:
\```rust
// 問題のあるコードスニペット
\```

**根拠**: `rules/project.md` — 「...該当規約の引用...」

**修正案**:
\```rust
// 修正後のコードスニペット
\```

---

### 🟡 [警告] ...

---

## 問題なし

変更内容のうち、規約に準拠している箇所があれば簡潔に記載する。
```

### 6. 完了報告

`notify_user` でレビュー完了を報告する。指摘件数のサマリーと出力ファイルパスを伝える。

## 注意事項

- **厳しさを保つこと**: レビューは「敵対的」に行う。「まあいいか」は禁止。
- **根拠を必ず示すこと**: すべての指摘に `rules/project.md` の該当箇所を明示する。
- **差分外のファイルには言及しない**: 変更されていないコードは対象外。
- **差分が存在しない場合**: ユーザーに「mainブランチとの差分がありません」と報告して終了。
