## Context

現在、季節イベント（資源生成、災害など）は `TurnService` 内にベタ書きされており、新しいイベントの追加やロジックの変更が困難です。これをリファクタリングし、拡張可能な形で「季節イベント」機能を独立させます。また、PRDの修正内容を反映し、洪水・疫病・反乱・資源生成の最新ロジックを実装します。

## Goals / Non-Goals

**Goals:**
- `SeasonalEventService` の導入による関心の分離。
- 洪水、疫病、反乱、資源生成（春・秋）、人口増加（春）の正確な実装。
- 忠誠度を「値」として扱う計算ロジック（表示値の 3〜5%）の正確な反映。
- イベント発生の結果をUI層に通知するためのデータ構造の定義。

**Non-Goals:**
- 戦争モードや外交ロジックの変更。
- 反乱による「領地の喪失」の実装（今回はリソース被害のみ）。

## Decisions

1. **`SeasonalEventService` の新設**
   - ターン開始時（災害・反乱）とターン終了時（生成・増加）の2つのタイミングでイベントを処理します。

2. **イベントロジックの詳細反映**
   - **人口増加**: 春 (`turn % 4 == 0`) 終了時に 10-12% 増加。
   - **資源生成**:
     - 春: 金が増加（忠誠度 3-5%、人口 10-15%、町 25-40%）。
     - 秋: 米が増加（忠誠度 3-5%、人口 10-15%、石高 25-40%）。
     - ※忠誠度の計算は、`Rate` の内部値ではなく「表示値（%）」に対してパーセンテージを適用する。
   - **洪水 (Flood)**: 夏 (`turn % 4 == 1`) 開始時に 1/40。被害大（石高・町 20-39%等）。
   - **疫病 (Plague)**: 季節問わず 1/40。被害（兵力 15-34%、人口 5-20%等）。
   - **反乱 (Rebellion)**: 忠誠度 < 50 の時、`(50 - 忠誠度)%` で発生。甚大な被害（兵力 30-50%等）。

3. **`Kuni` モデルの更新**
   - リソースを割合で減少させる `apply_percentage_loss(ResourceSelector, Percentage)` のようなメソッド、または各災害専用のメソッドを実装し、DomainError を回避しながら正確に減少させます（saturating_sub 等の活用）。

4. **イベント通知データ構造**
   - `SeasonalEventEffect` を更新し、被害項目のリストを詳細に持てるようにします。

## Risks / Trade-offs

- **反乱の連鎖**: 忠誠度が低いと反乱が発生し、被害でさらに忠誠度が下がるため、詰み状態になるリスクがあります。UIでの警告表示が重要になります。
- **計算精度**: パーセンテージ計算が重なるため、丸め誤差や `Amount` 型（u32）への変換時の扱いを慎重に行います。
