## Context

現在の `Kuni` ドメインモデルにおける内政コマンドの計算式は、`Amount` 型（10倍の内部スケールを持つ）と `Rate` 型（0-100の整数値）の間で単位の不整合が発生している。
これにより、忠誠度（`tyu`）の増減が本来の意図よりも10倍大きくなってしまっている。

## Goals / Non-Goals

**Goals:**
- 内政コマンド（雇用、解雇、施し）による忠誠度の増減を、表示単位に基づいた適切なスケールに修正する。
- PRDの「資源計算式」に基づきつつ、`Rate` 型の範囲（0-100）に適合させる。

**Non-Goals:**
- 資源（金、米、兵、人口）自体の計算ロジックの変更。
- 新しい内政コマンドの追加。

## Decisions

### 1. 表示単位 (`DisplayAmount`) と内部単位 (`Amount`) の分離
- **Decision**: ユーザー入力や表示に用いる `DisplayAmount` と、計算に用いる `Amount` を別の型として定義する。
- **Rationale**: これまでの実装では `Amount` 型が「内部値（BIAS倍）」と「表示値（整数）」の両方の文脈で使われており、計算ミス（二重のBIAS適用やBIAS適用忘れ）の温床となっていた。型レベルで分離することで、コンパイル時にこれらの誤りを防ぐ。

### 2. 内部スケール (`INTERNAL_SCALE`) の変更 (10 -> 100)
- **Decision**: `INTERNAL_SCALE` を 100 に変更する。
- **Rationale**: 10倍（BIAS=10）では、金から米への換算などで精度が不足する場合がある。100倍（BIAS=100）にすることで、より柔軟な計算式（パーセンテージベースの計算など）を精度良く扱えるようにする。

### 3. 徴募（雇用）時の忠誠度減少
- **Before:** `tyu_loss = amount.value() / 2` (投入量が表示10の場合、raw 100 / 2 = 50減少 ※旧BIAS=10)
- **After:** `tyu_loss = amount.to_display().value() / 2` (投入量が表示10の場合、10 / 2 = 5減少)
- **Rationale:** 忠誠度は0-100の範囲であり、10ユニットの兵を雇うだけで50ポイント減少するのは過大である。表示単位（1ユニット = 0.5ポイント減少）に合わせることでバランスを改善する。

### 4. 解雇時の忠誠度上昇
- **Before:** `tyu_gain = amount.value() / 2`
- **After:** `tyu_gain = amount.to_display().value() / 2`
- **Rationale:** 雇用と同様に、表示単位ベースでの上昇に修正。

### 5. 施し時の忠誠度上昇
- **Before:** `tyu_gain = (amount.value() * (bias_half + rng)) / INTERNAL_SCALE`
- **After:** `tyu_gain = (amount.to_internal().mul_percent(rng))` (rng: 50..100)
- **Rationale:** 10の米を施すことで忠誠度が5-9ポイント上昇するのが妥当である。

## Risks / Trade-offs

- **低投入時の切り捨て:** 表示単位が1の場合、上昇/減少が 0 になる可能性がある（例：1/2 = 0）。これはオリジナルのゲームバランスでも見られる挙動であり、許容範囲とする。より細かな制御が必要な場合は、忠誠度の内部表現を10倍にする等の対応が必要だが、今回は既存の `Rate` 型の仕様を維持する。
