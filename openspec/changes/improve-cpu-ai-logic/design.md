## Context

現在の `CpuActionDecisionService` は自身の国のステータスのみを考慮し、かつランダムにアクションを選択しています。また、戦闘ロジックはプレイヤー対戦を前提としたターン制の策選択のみが実装されており、CPU同士の対戦も同様の処理を通るため計算上のオーバーヘッドと冗長なログが発生しています。

## Goals / Non-Goals

**Goals:**
- 近隣諸国の兵力に基づいた動的な意思決定（比較優位性の評価）の実装。
- 兵力 > 金 > 米の優先順位に基づくリソース管理。
- プレイヤーの戦術傾向に応じたカウンター策の選択。
- CPU同士の戦争における確率的な即時解決。
- MCP経由でのAI思考プロセスの可視化と制御。

**Non-Goals:**
- 長期的な同盟関係の構築や外交ロジック（本タスクではスコアリングのみ）。
- 複雑な経路探索や遠隔地への遠征（隣接国のみを対象）。

## Decisions

### 1. AI意思決定ロジック (Scoring System)
`CpuActionDecisionService` に `KuniRepository` と `NeighborRepository` を注入（または `decide` メソッドに近隣データリストを渡す）するように変更します。

各ターンのスコアリング計算：
- **Defense Score**: `自軍兵力 / MAX(隣接敵国の兵力)`. 1.2未満で「雇用」の重みを最大化。
- **Offense Score**: `自軍兵力 / MIN(隣接敵国の兵力)`. 1.5以上かつ資源十分で「戦争」の重みを増加。
- **Economy Score**: 維持費に対する資源残量。不足分に応じて「開発」「米売り買い」を重み付け。

最終的な行動は、これらのスコアを重みとした乱数選択で行います。

### 2. CPU vs CPU 戦争の解決
`BattleService` に `resolve_cpu_vs_cpu` メソッドを追加。
- 勝率 $P = \frac{Atk^2}{Atk^2 + Def^2}$ を基本とし、士気や兵糧の状態を補正値として加味。
- 1回の計算で決着をつけ、占領処理または防衛成功処理を実行。

### 3. 戦術カウンターロジック
`PlayerTacticHistory` エンティティ（または `GameState` 内のメタデータ）を導入。
- プレイヤーの直近10回の戦術を記録。
- 戦闘開始時にCPUがプレイヤーの「癖」を読み、PRDの戦術組み合わせ表に基づいた有利な策を確率的に選択。

### 4. アクションログの拡張
`DomesticLogEvent::CpuAction` に `reasoning: Option<String>` を追加し、UIやMCPから「なぜその行動を選んだか」を表示可能にします。

### 5. MCPデバッグツールの追加
- `set_kuni_resource`: テスト用の資源セット。
- `advance_turns`: 複数ターンの自動実行。
- `get_ai_analysis`: 特定の国の全隣接国比較スコアの表示。

## Risks / Trade-offs

- **計算コスト**: 毎ターン近隣諸国の情報を取得するため、DBアクセスが増加します。キャッシュまたはバルク取得で最適化します。
- **バランス調整**: スコアリングの重みが不適切だと、特定の大名が異常に強くなったり、全員が引きこもったりする可能性があります。MCPツールでの継続的な微調整が必要です。
