# Change: PRDに基づくドメインロジックの実装

## Why
`PRD.md` では「戦国統一」のコアとなるメカニクス（戦争モード、内政モード、資源管理、ターン進行イベント）が定義されています。この変更では、`project.md` のクリーンアーキテクチャ・ガイドラインに従い、これらの機能をサポートするために必要なドメインロジック（モデル、サービス、ユースケース）を実装します。

## What Changes
- `Daimyo`（大名）、`Kuni`（領地）、資源などのバリューオブジェクトおよびエンティティを実装。
- ターン進行、季節イベント、戦闘計算のドメインサービスを実装。
- 全ての内政コマンド（米売り、米買い、開墾、町造り、雇用、解雇、施し、輸送、委任・解任、情報）および全ての戦争コマンド（通常、奇襲、火計、鼓舞）のアプリケーションユースケースを実装。
- ゲーム状態の保存・読み込みを行うためのリポジトリインターフェースを定義。

## Impact
- 影響を受ける仕様 (Affected specs): `manage-domestic`, `execute-battle`, `progress-turn`
- 影響を受けるコード: `crates/engine/src/domain/`, `crates/engine/src/application/`
