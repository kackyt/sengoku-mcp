## Context

現在の合戦システム（戦争モード）は、兵力と食料の投入、および戦闘中の戦術選択のループが未実装です。
特に兵糧（米）の消費ロジックはドメインサービス層には存在するものの、アプリケーション層からの入力や管理が行われていません。
本設計では、出陣時の資源選択フローと、戦闘決着までのインタラクティブなループを導入します。

## Goals / Non-Goals

**Goals:**
- 出陣時に兵数と米の量をユーザーが任意に指定できるようにする。
- 合戦中の兵力、食料、士気の変動を可視化する。
- 1ターンごとに戦術を選択し、決着（または退却）まで合戦を継続させる。
- 兵糧切れや士気喪失による敗走判定を正しく動作させる。

**Non-Goals:**
- 複数日にわたる包囲戦や、マップ上での軍勢移動の追加（本件は隣接国間戦闘の完結に集中）。
- 資金（金）を戦闘中に直接消費する特殊戦術の追加（将来的な拡張とする）。

## Decisions

1. **出陣前入力フローの拡張**:
   - `ScreenState::Domestic` の `sub_state` に `InputWarResources` を追加し、兵数入力 -> 米入力の順で遷移させます。
   - 確定時に `BattleUseCase::start_war` を呼び出し、出撃元の国から資源を減らした上で合戦画面に遷移します。

2. **合戦ステータスの管理**:
   - 攻撃軍のステータスを `WarState` 構造体として定義し、`ScreenState::War` で保持します。
   - 兵力、米、士気の3要素を管理し、毎ターンの `BattleUseCase::execute_battle_turn` で更新します。

3. **合戦ループの制御**:
   - `cli::handler` において、`WarSubState::SelectTactic` で戦術を選んだ後、`BattleResult` を受け取り、決着がついていない場合は再び策選択画面に戻します。
   - `Tactic::Retreat`（退却）が選ばれた場合は、即座に合戦を終了し内政画面に戻ります。

4. **ドメインモデルの調整**:
   - `BattleService` が「士気」として利用している値を、国の `tyu`（忠誠度）とは別に「軍勢の士気」として扱えるように `BattleUseCase` 側でラップします。

## Risks / Trade-offs

- **UIの複雑化**: 入力ステップが増えるため、ユーザーの操作回数が増加します。デフォルト値を最大兵力の半分などに設定することで入力を補助します。
- **データ整合性**: 合戦中にアプリが終了した場合の資源の扱い（出撃時に減らしているため、消失する可能性がある）。今回は自動セーブの仕組みに乗ることでリスクを最小化します。
